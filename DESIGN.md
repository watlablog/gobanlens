# DESIGN.md — GobanLens（碁盤レンズ）設計メモ（MVP）

## 0. 概要
**GobanLens** は、スマホ/PCのカメラや写真から囲碁の盤面（1局面）を取り込み、**盤面をSGFとして再構成**し、必要に応じて **地合い推定**や **AI（KataGo）による診断/次の一手**を表示するツールです。  
詰碁本の局面を撮影して解析する用途も想定します。

> 本ドキュメントは「設計思想」「目的」「実装手段アイデア」「進め方（ステップ分割）」を、MVP目線で簡潔にまとめます。

---

## 1. 目的 / ゴール

### 1.1 ユーザー価値
- 盤面写真を撮るだけで、**局面をデジタル化**して見返せる（SGF化）
- AIにより **良い手の候補**を学べる（検討の入口）
- 詰碁本の局面を取り込んで **解答/ヒント**として使える

### 1.2 MVP（最小実用）で実現すること
- 画像入力（撮影 or アップロード）
- 盤面領域の確定（手動補助あり）→射影変換（透視補正）
- 交点ごとの石（黒/白/空）推定 → SGF（AB/AW）生成
- OSS碁盤UI（例：WGo.js）で局面を表示
- （オプション）KataGo解析で最善手/上位候補を表示

### 1.3 MVPでやらないこと（後回し）
- 手順追跡（映像から棋譜を進める、安定化、差分確定、分岐など）
- 手や障害物の自動除外など高度なエラーハンドリング
- 地合い計算ロジックの自作（既存OSS/外部に寄せる方針）
- ブラウザ内でのKataGo推論（WebGPU/WASM最適化は後段）

---

## 2. 設計思想（重要）

### 2.1 「1局面」特化で難所を捨てる
時系列（1手ずつ追う）を捨てることで、以下の難所を避ける：
- 誤検出の蓄積、安定化、コウ/取り込み整合、分岐管理
- 「手が写っているとき無視する」などの複雑な制御

→ MVPでは **1枚の写真を確実に局面化**することを最優先にする。

### 2.2 失敗をUIで救済する（自動化は後から）
盤面検出や石推定は100%自動化が難しいため、MVPは次を許容する：
- 盤の四隅（4点）をユーザーが調整
- 石の誤判定はタップで修正（黒→白→空トグル）
- 「黒白反転」「回転/左右反転」などの救済ボタン

→ まず “使える” を作り、後から自動化精度を上げる。

### 2.3 コアはOSS、課金はサービスで
- 盤面→SGF→表示のコアはOSSで成立させる
- 後で課金するなら、主に以下をサービス化する：
  - AI解析回数/強度（visits）
  - 履歴同期、共有、チーム、クラウド保存
  - 高解像度/大量処理などの上位機能

## 3. アーキテクチャ案（推奨）

### 3.1 フロント（Web/PWA）— コア
- **入力**：カメラ撮影 / 画像アップロード
- **射影変換**：OpenCV.js 等（WASM）
- **石推定**：交点周辺パッチの簡易分類（MVP）
- **表示**：WGo.js 等（局面表示・候補手オーバーレイ）
- **出力**：SGF生成（AB/AW, PL）・共有/ダウンロード

### 3.2 バックエンド（Python）— AI解析（オプション）
- FastAPI などで `POST /analyze` を提供
- 入力：SGF（または盤配列）+ 手番 + コミ
- 出力：最善手 + 上位候補 + 付随情報（勝率/目差/PV）
- 実行：KataGo をサーバ側で実行
- デプロイ：Cloud Run など（コンテナで管理しやすい）

### 3.3 BaaS（必要になったら）
- 認証・課金・保存（Firebase Auth / Firestore / Stripe 等）
- MVPでは最初から入れず、後段で段階導入する（実装の独立性を保つ）

## 4. MVP画面遷移（最小4画面）

### 0. Home（入力選択）
- 撮影する / 画像を選ぶ
- 設定（任意）：盤サイズ（9/13/19）、次に打つ番（黒/白）、コミ

### 1. Crop（盤の四隅を確定）
- 四隅4点をドラッグで調整（MVPの基本）
- 自動検出（オプション、失敗時は手動）
- 次へ（射影変換）

### 2. Verify（石推定の確認・修正）
- 盤画像プレビュー + WGo盤面（推定結果）
- 誤りはタップで修正（黒→白→空）
- 黒白反転、回転/左右反転（救済）
- 解析する（KataGo）ボタン（オプション）

### 3. Analyze（候補手表示）
- 最善手を強調、上位候補を番号付きで表示
- （拡張）この手を打って再解析（詰碁トレーナー入口）
- 別画像で解析 / SGF保存・共有

---

## 5. 実装手段のアイデア（MVP向け）

### 5.1 盤面の正規化（射影変換）
1. 入力画像に対してユーザーが四隅4点を指定
2. 透視変換して正方形盤面へワープ
3. 盤サイズ（19×19等）に応じて交点座標を生成（正規化座標）

### 5.2 石（黒/白/空）推定（MVP）
- 交点近傍のパッチを切り出し、以下の簡易特徴で分類：
  - 輝度（黒/白）
  - 局所コントラスト（石の輪郭）
  - 彩度（盤面木目との差）
- 最初は誤判定をUI修正で吸収し、精度向上は後段で実施

### 5.3 SGF生成（1局面）
- 手順を作らず、**AB/AW** で局面を表現
- 詰碁用途のため、**次に打つ番（PL）** を設定できるようにする
- 出力はダウンロード/共有（テキストファイル）でOK

### 5.4 AI解析（オプション）
- サーバでKataGo解析（Cloud Run + FastAPI）
- 最小I/F（案）
```json
POST /analyze
{
  "sgf": "(;GM[1]FF[4]SZ[19]PL[B]AB[dd]AW[pp])",
  "komi": 6.5,
  "visits": 200,
  "max_moves": 5
}
```
```json
{
  "bestMove": "qd",
  "candidates": [
    {"move":"qd","winrate":0.62,"scoreLead":3.1,"pv":["qd","dp","cq"]}
  ]
}
```

## 6. 進め方（ステップ分割）

目的：依存関係が絡む機能（ログイン、カメラ、AI解析）を分離し、**常に動く状態を保ちながら**積み上げる。

### Step 1 — ローカル画像→局面表示（最小の核）
- 画像アップロードのみ（カメラは後回し）
- 四隅4点調整 → 射影変換
- 交点オーバーレイを表示（位置が合うか確認）
- 盤面は最初は手入力でもOK（UI骨格を先に作る）

成果物：画像を読み込み、盤の座標系が取れる。

### Step 2 — 石推定（簡易）＋修正UI
- 交点ごとに黒/白/空を自動推定（荒くてOK）
- WGo盤面に反映
- タップ修正（黒→白→空）と「黒白反転」ボタンを実装

成果物：実写真から「それっぽい局面」が作れ、修正で正しくできる。

### Step 3 — SGF生成・保存（AB/AW + PL）
- 現在の盤面からSGFを生成
- ダウンロード（.sgf）/クリップボードコピー

成果物：局面が“データ”として外に出せる（最重要の完成感）。

### Step 4 — カメラ入力（ブラウザ getUserMedia）
- カメラ撮影→静止画キャプチャ（1枚）に限定
- Step1〜3の入力を「カメラキャプチャ画像」に差し替え

成果物：スマホで撮ってその場で局面化できる。

### Step 5 — AI解析（サーバ）
- Python API（FastAPI）を作成し、KataGo解析を返す
- フロントで解析結果を盤にオーバーレイ（最善手＋上位候補）
- まずはローカル実行→次にCloud Runへ

成果物：AIによる“次の一手/診断”が成立。

### Step 6 — 詰碁モード（軽い拡張）
- 「ユーザーが1手打つ」→再解析→評価
- ヒント（上位候補を薄く表示）

成果物：詰碁本に強い学習ツールに進化。

### Step 7 — ログイン/課金/クラウド保存（別実装として追加）
- Auth（Firebase等）
- 保存（局面・解析結果・学習履歴）
- 課金（解析回数/高精度解析/同期等）

成果物：プロダクト化（運用・収益化）に必要な部分を追加。
